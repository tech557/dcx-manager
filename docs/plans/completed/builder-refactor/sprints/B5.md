## B5.1 — Editor session state model

**Requirements:** BLD-EDT-001, BLD-EDT-002  
**Files to inspect:** `src/builder/islands/EditorViewerIsland/useEditorPanel.ts`, `src/builder/stage/stageContext.types.ts`  
**Files to change:** `src/builder/islands/EditorViewerIsland/useEditorPanel.ts`

**Required behaviour:**
```typescript
interface EditorSession {
  taskId: string;
  isMinimized: boolean;
  hasDraft: boolean;
  openedAt: number;
}
// State: sessions[], activeTaskId
// openSession: if exists → restore; if > 5 open → session pill area becomes horizontally scrollable, never replace or discard an existing session ✅
// minimizeSession: set isMinimized=true
// closeSession: if hasDraft → require explicit discard; remove session
// switchSession: set isMinimized=false + setActiveTaskId
```
No hard product limit in V1; display up to 5 session pills directly, with horizontally scrollable overflow when more than 5 are open (✅ BLD-EDT-002 / OD-001). Unsaved sessions NEVER silently replaced ✅.

**Acceptance criteria:**
```
□ Opening 6th session expands pills into horizontally scrollable overflow, existing sessions unchanged ✅
□ Minimizing preserves draft state
□ Closing with hasDraft=true requires explicit discard
□ npm run typecheck passes
```

## B5.2 — Session pill UI

**Files to change:** `src/builder/islands/EditorViewerIsland/EditorSessionPill.tsx` (CREATE), `src/builder/islands/EditorViewerIsland/EditorViewerIsland.tsx` (EDIT)

**Required behaviour:** For each minimized session: render a round pill above/beside the collapsed editor icon. Pill shows first char of task name + readiness colour dot. Clicking pill calls switchSession(). When all sessions minimized: editor returns to 4.5rem. When active session exists: editor expands to 25rem.

**Acceptance criteria:**
```
□ EditorSessionPill.tsx ≤ 50 lines
□ Minimise → pill appears, restore → session intact
□ npm run typecheck passes
```
