## B6.1 — Presentation mode engine

**Requirements:** BLD-SLC-001  
**Files to inspect:** `src/builder/islands/SelectionIsland/SelectionIsland.tsx`, `src/builder/stage/StageProvider.tsx`, `src/builder/stage/stageContext.types.ts`  
**Files to change:** `src/builder/islands/SelectionIsland/usePresentationMode.ts` (CREATE), `src/builder/stage/StageProvider.tsx` (add isPresentationActive, enterPresentationMode, exitPresentationMode), `src/builder/stage/stageContext.types.ts`, `src/builder/islands/SelectionIsland/SelectionLabel.tsx`

**Required behaviour:** `enterPresentationMode(selectedId)`: store current `expandedNodeIds`, find ancestors, set expandedNodeIds = ancestors + selected, set `isPresentationActive=true`, scroll selected card into view. `exitPresentationMode()`: restore expandedNodeIds from saved snapshot, set `isPresentationActive=false`. V1: single selection only ⏱. In SelectionLabel: when `selectedNodeIds.length === 1` and not in presentation mode → count is a clickable button.

**Acceptance criteria:**
```
□ Single card selected → clicking count enters presentation mode
□ Unrelated cards collapse, ancestors + selected card expanded
□ Exiting restores previous expanded states exactly
□ Multiple selected: no presentation button shown
□ npm run typecheck passes
```
